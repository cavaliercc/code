# PLAN

## 1. 目标能力范围
- 大规模手写识别
- 数学公式精确还原
- 古籍 / 特殊排版识别
- 云端协同与多人标注

## 2. 模型与推理架构
采用“快路径 + 难例升级”的本地推理体系：

### Lite Path
默认识别链路使用 **PP-OCRv5 + PP-StructureV3**，负责：
- 文本检测
- 文本识别
- 版面分析
- 表格恢复

### Pro Path
复杂文档使用 **PaddleOCR-VL-1.5**，重点处理：
- 多栏排版
- 嵌套表格
- 图片穿插文本
- 低清扫描
- 异常阅读顺序

### 路由策略
- 默认先走 Lite Path。
- 当检测到以下情况时，自动升级到 Pro Path，并支持页级或区域级重跑：
  - 复杂布局
  - 表格恢复失败
  - 字段冲突
  - 低置信分段

### 模型安装策略
- 默认提供 CPU 可用轻量包。
- Pro 模型按需下载/导入。
- 检测到 GPU 时自动启用高性能推理。

## 3. 自学习 / 自我进化设计
首版即支持“反馈学习”，并为“本地微调”预留。

### Correction Memory
记录并沉淀：
- 用户纠错前后值
- 坐标与页区域
- 上下文
- 文档类型
- 原始置信度

### Layout Template Learning
对高频文档做版式聚类，学习稳定结构：
- 标题区
- 正文区
- 表格区
- 页眉页脚
- 字段区域

### Domain Lexicon
构建本地词典，覆盖：
- 公司名
- 专业术语
- 产品名
- 缩写
- 计量单位
- 表头词
- OCR 易错词

### Post-OCR Resolver
在原始 OCR 后追加纠错层，结合：
- 模板
- 词典
- 上下文
- 历史反馈

对候选结果进行重排序。

### Adaptive Engine Policy
对高频出错文档类型自动提升模型等级，或切换到更合适链路。

### Local Fine-tune Pipeline（二阶段）
支持本地增量训练：
- 将用户确认过的纠错样本沉淀为训练集
- 对特定文档类型执行 LoRA / 增量微调

微调触发策略：
- 不实时训练
- 手动触发或周期性后台任务
- 仅对达到样本阈值的文档簇执行

安全约束：
- 学习结果按文档簇/模板生效
- 防止一次错误反馈污染全部文档

## 4. 中间结构与导出
所有引擎输出先归一为统一文档结构，再执行导出，避免与单一模型强绑定。

### 统一结构建议
- Document
- Page
- Block
- Paragraph
- Table
- Cell
- FigureRegion
- ReadingOrder
- Confidence
- CorrectionTrace

### 导出策略
- **txt**：输出纯文本阅读顺序结果
- **md**：尽量保留标题、段落、列表、表格；表格输出为 Markdown table；无法稳定恢复时回退文本块
- **docx**：保留段落、标题、分页顺序、基础表格结构和图文顺序
- **xlsx**：以表格为核心输出；一个文档可拆多个 sheet；非表格文档可选输出“结构化块索引”sheet

### 导出模式
- 版面还原优先
- 结构化提取优先

## 5. 跨平台工程方案
建议固定如下实现方式，避免后续反复改架构：

- UI：**Tauri 2 + React**
- 本地服务：**FastAPI**
- 推理运行时：Python 环境独立封装
- 文档处理模块化：PDF 渲染、图像预处理、OCR 编排、导出分离

### 平台支持
- Windows x64
- macOS Intel / Apple Silicon
- Linux x64

### 安装方式
- 桌面应用安装包 + 首次引导下载模型

### 本地目录规范
- `models/`
- `workspace/`
- `results/`
- `exports/`
- `learning/`
- `logs/`

## 6. 内部接口定义（Public APIs / Interfaces）
### RecognizeRequest
- 文件路径
- 语言
- 文档类型偏好
- 性能档位
- 导出偏好
- 是否启用学习增强

### RecognizeResult
- 标准化文档结构
- 原始 OCR 文本
- 表格
- 阅读顺序
- 置信度
- 引擎链路

### FeedbackRecord
- 用户纠错内容
- 影响范围
- 模板归属
- 是否允许用于训练

### EngineDecision
- 当前任务走 Lite 或 Pro 的原因
- 是否进行了升级重跑

### ExportRequest
- 目标格式
- 导出模式
- 表格拆分策略
- 是否附带元数据

### LearningJob
- 模板学习
- 词典更新
- 规则更新
- 本地微调任务统一入口

## 7. Test Plan（验收覆盖）
必须覆盖以下场景：

1. 中文/英文扫描 PDF、合同、报告、说明书、图文混排文档
2. 普通表格、复杂表格、跨页表格、合并单元格表格
3. 清晰扫描、拍照倾斜、模糊、阴影、压缩失真、彩色背景
4. CPU-only 环境可识别并完成四种格式导出
5. GPU 环境复杂文档可自动升级到 Pro Path 并提升结果
6. 用户纠错后，相似版式文档可自动改善标题、表头、段落顺序或术语识别
7. 局部错误反馈不会错误泛化到不相关文档
8. docx/xlsx/txt/md 四类导出文件可直接打开且结构完整
9. 大文件、多页 PDF、批量导入、任务取消、恢复、模型缺失、模型升级、导出失败重试
10. 完全离线状态下功能可闭环运行

## 8. 首版指标建议
- 通用中英文档文本块级准确率：**>= 95%**
- 常见表格结构恢复可用率：**>= 90%**
- 高频模板文档在学习后关键结构识别提升：**>= 5%**
- CPU 模式保证可用，GPU 模式显著提升复杂页处理效率

## 9. Assumptions
- 首版是桌面 GUI，而非浏览器 SaaS
- 首版语言范围为中文 + 英文
- 首版重点从“票据/单据/表单”调整为“通用文档 + 表格”
- 首版学习机制包含：
  - 反馈驱动优化（必做）
  - 本地微调能力（二阶段，并在架构上预留）
- 首版主路线优先选择 Paddle 生态，确保跨平台、本地部署成熟度与工程可控性
