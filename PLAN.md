# 本地化 OCR 桌面软件项目计划

## Summary
构建一个完全本地运行的跨平台 OCR 桌面应用，首版优先覆盖中文/英文的通用文档与表格场景，支持 Win/macOS/Linux，支持导出 `docx`、`xlsx`、`txt`、`md`。
核心目标不是单次识别，而是形成"识别 -> 用户纠错 -> 本地学习 -> 后续自动优化"的闭环，并预留本地增量训练能力。

模型路线采用"双层方案"：

- 默认生产主路线：`PaddleOCR 3.x / PP-OCRv5 + PP-StructureV3`
 优点是本地部署成熟、跨平台友好、CPU 可跑、表格和版面恢复能力强，最适合首版落地。
- 增强大模型路线：`PaddleOCR-VL-1.5`
 用于复杂版面、低置信页面、图文混排、跨区域结构理解与高难样本重识别。
- 备选但不作为首版主路线：
 `MinerU` 适合 PDF/学术文档深度解析，可作为二阶段增强；
 `GOT-OCR2.0` 泛化强，但本地桌面产品化与资源友好度不如 Paddle 路线。

产品形态建议固定为：

- 桌面端：`Tauri 2 + React`
- 本地推理服务：`Python FastAPI`
- 本地数据：`SQLite + 文件系统`
- 所有识别、学习、导出默认离线完成

## Key Changes
### 1. 产品能力边界
首版聚焦以下输入和输出：

- 输入：
 图片、扫描件、PDF、截图、拍照文档
- 重点识别对象：
 合同、报告、表格、说明书、扫描 PDF、图文混排办公文档
- 输出：
 `docx`、`xlsx`、`txt`、`md`
- 学习目标：
 提升高频文档类型、固定版式文档、常见专业术语、表头/字段/段落结构的识别质量

非首版重点：

- 大规模手写识别
- 数学公式精确还原
- 古籍/特殊排版
- 云端协同与多人标注

### 2. 模型与推理架构
采用"快路径 + 难例升级"的本地推理体系：

- `Lite Path`
 使用 `PP-OCRv5 + PP-StructureV3` 作为默认识别链路，负责文本检测、识别、版面分析、表格恢复。
- `Pro Path`
 使用 `PaddleOCR-VL-1.5` 处理复杂文档：
 多栏排版、嵌套表格、图片穿插文本、低清扫描、异常阅读顺序。
- 路由策略：
 默认先走 Lite；
 当检测到复杂布局、表格恢复失败、字段冲突、低置信分段时，自动升级页级或区域级重跑。
- 模型安装策略：
 默认提供 CPU 可用轻量包；
 Pro 模型按需下载/导入；
 GPU 存在时自动启用高性能推理。

### 3. 自学习 / 自我进化设计
你要求长期自优化，因此首版应同时支持"反馈学习"和"本地微调预留"。

- `Correction Memory`
 记录用户纠错前后值、坐标、页区域、上下文、文档类型、原始置信度。
- `Layout Template Learning`
 对高频文档做版式聚类，学习标题区、正文区、表格区、页眉页脚、字段区域的稳定结构。
- `Domain Lexicon`
 构建本地词典：
 公司名、专业术语、产品名、缩写、计量单位、表头词、常见 OCR 易错词。
- `Post-OCR Resolver`
 在原始 OCR 后追加纠错层，结合模板、词典、上下文和历史反馈，对候选结果重新排序。
- `Adaptive Engine Policy`
 对经常出错的文档类型自动提升模型等级或切换更适合的处理链路。
- `Local Fine-tune Pipeline`
 二阶段支持本地增量训练：
 将用户确认过的纠错样本沉淀为训练集，支持对特定文档类型做 LoRA/增量微调。
- 微调触发策略：
 不实时训练；
 采用手动触发或周期性后台任务；
 仅对达到样本阈值的文档簇执行。
- 安全约束：
 学习结果按文档簇/模板生效，避免一次错误反馈污染全部文档。

### 4. 中间结构与导出
所有引擎输出必须先归一为统一文档结构，再导出，避免被单一模型绑死。

统一结构建议：

- `Document`
- `Page`
- `Block`
- `Paragraph`
- `Table`
- `Cell`
- `FigureRegion`
- `ReadingOrder`
- `Confidence`
- `CorrectionTrace`

导出策略固定如下：

- `txt`
 输出纯文本阅读顺序结果。
- `md`
 尽量保留标题、段落、列表、表格；
 表格输出为 Markdown table；
 无法稳定恢复时回退文本块。
- `docx`
 保留段落、标题、分页顺序、基础表格结构和图文顺序。
- `xlsx`
 以表格为核心输出；
 一个文档可拆多个 sheet；
 非表格文档可选输出"结构化块索引"sheet。
- 导出模式提供两种：
 `版面还原优先`
 `结构化提取优先`

### 5. 跨平台工程方案
实现方式建议固定，避免后续反复改架构：

- UI：`Tauri 2 + React`
- 本地服务：`FastAPI`
- 推理运行时：Python 环境独立封装
- 文档处理：
 PDF 渲染、图像预处理、OCR 编排、导出分别模块化
- 平台支持：
 Win x64
 macOS Intel / Apple Silicon
 Linux x64
- 安装方式：
 桌面应用安装包 + 首次引导下载模型
- 本地目录：
 `models/`
 `workspace/`
 `results/`
 `exports/`
 `learning/`
 `logs/`

## Public APIs / Interfaces
需要先定义清楚的内部接口：

- `RecognizeRequest`
 文件路径、语言、文档类型偏好、性能档位、导出偏好、是否启用学习增强。
- `RecognizeResult`
 标准化文档结构、原始 OCR 文本、表格、阅读顺序、置信度、引擎链路。
- `FeedbackRecord`
 用户纠错内容、影响范围、模板归属、是否允许用于训练。
- `EngineDecision`
 当前任务为何走 Lite 或 Pro，是否进行了升级重跑。
- `ExportRequest`
 目标格式、导出模式、表格拆分策略、是否附带元数据。
- `LearningJob`
 模板学习、词典更新、规则更新、本地微调任务的统一入口。

## Test Plan
必须覆盖这些验收场景：

- 中文/英文扫描 PDF、合同、报告、说明书、图文混排文档。
- 普通表格、复杂表格、跨页表格、合并单元格表格。
- 清晰扫描、拍照倾斜、模糊、阴影、压缩失真、彩色背景。
- CPU-only 环境可识别并完成四种格式导出。
- GPU 环境复杂文档可自动升级到 Pro 路径并提升结果。
- 用户纠错后，相似版式文档能自动改善标题、表头、段落顺序或术语识别。
- 局部错误反馈不会错误泛化到不相关文档。
- `docx/xlsx/txt/md` 四类导出文件可直接打开且结构完整。
- 大文件、多页 PDF、批量导入、任务取消、恢复、模型缺失、模型升级、导出失败重试。
- 完全离线状态下功能可闭环运行。

建议首版指标：

- 通用中英文档文本块级准确率：`>= 95%`
- 常见表格结构恢复可用率：`>= 90%`
- 高频模板文档在学习后关键结构识别提升：`>= 5%`
- CPU 模式保证可用，GPU 模式显著提升复杂页处理效率

## Assumptions
- 产品首版是桌面 GUI，而不是浏览器 SaaS。
- 首版语言范围为中文+英文。
- 首版重点从"票据/单据/表单"修正为"通用文档+表格"。
- 首版学习机制包括：
 反馈驱动优化为必做；
 本地微调能力作为二阶段设计并在架构上预留。
- 首版主路线优先选 Paddle 生态，以保证跨平台、本地部署成熟度和工程可控性。
